## 概述

HMI Asset Studio 是一款面向智能座舱 HMI 开发团队的 Unity Editor 工具，用于统一组织材质、模型、特效与场景模板等核心资产。它提供可视化浏览、批量操作、场景搭建和车辆配置等一站式能力，帮助团队快速理解当前内容构成并高效推进开发。

### 核心能力

| 功能模块 | 说明 |
|---------|------|
| 资产浏览 | 按材质、模型、特效分类浏览，支持网格/列表切换和缩放 |
| 批量替换 | 选中目标后预览候选材质，一键批量替换 |
| 场景搭建 | 基于模板选择照明、相机、天气、地面、天空，一键生成场景 |
| 车辆配置 | 导入 FBX，解析零件结构，绑定材质并导出 Schema |
| 材质对比 | A/B 双栏对比材质属性差异 |

## 环境要求

### 必需

- **Tuanjie Engine** (Unity China) 2022.3 或更高版本
- **HMIRP Core** (`com.unity.render-pipelines.universal`) >= 1.0.3

### 推荐安装（完整功能）

安装以下四个 HMIRP 包后，所有功能模块均可使用：

| 包名 | 显示名 | 最低版本 | 影响范围 |
|------|--------|---------|---------|
| `com.unity.render-pipelines.universal` | HMIRP Core | 1.0.3 | 材质预览、场景搭建、批量替换 |
| `com.tuanjie.hmirp.shaderlibrary` | HMIRP Shader Library | 1.0.3 | Shader 浏览与替换 |
| `com.tuanjie.hmirp.materiallibrary` | HMIRP Material Library | 1.0.4 | 材质库内容 |
| `com.tuanjie.render-pipelines.hmirp.staterendersystem` | HMIRP State Render | 1.0.4 | 高级状态效果 |

> 支持通过 git URL、本地 file: 路径或 Package Manager 安装，工具会自动识别版本。

### 运行模式

- **完整模式**：四包全部就绪，所有功能可用
- **预览模式**：部分包缺失，可用功能降级（如仅支持资产浏览和车辆配置）

## 快速开始

### 1. 安装 Package

将 `com.hmi.workspace` 添加到项目的 `Packages/manifest.json`：

```json
"com.hmi.workspace": "file:路径/TuanjieHMIAssetWorkspace"
```

### 2. 打开欢迎页

菜单栏 → **Window → HMI Asset Studio**

### 3. 进入工作区

在欢迎页点击 **"打开工作区"** 按钮，或通过菜单栏 → **Window → HMI Asset Workspace** 直接进入。

### 4. 查看环境状态

进入工作区后，首页会自动检测已安装的 HMIRP 包并显示：
- 资产加载数量
- 环境就绪状态
- 可用能力数量（x/4）

根据首页推荐的下一步操作开始使用。

## 欢迎页

**入口：** `Window → HMI Asset Studio`

欢迎页是独立的 EditorWindow，作为 HMI Asset Studio 的入口和介绍页面。

### 页面结构

- **Hero 区域** — 产品标题、简介和核心操作按钮，背景展示产品视觉
- **能力卡片** — 三张卡片介绍平台定位、能力范围和使用路径
- **底部链接** — 文档中心、示例工程、更新记录、技术支持

### 操作

| 按钮 | 功能 |
|------|------|
| 打开工作区 | 关闭欢迎页，打开 HMI Asset Studio 主工作区 |
| 阅读文档 | 在系统文件管理器中打开本文档所在目录 |

## 工作台首页

进入工作区后默认显示首页仪表盘。

### 环境状态栏

显示三项关键指标：
- **资产计数** — 已扫描并加载的资产总数
- **依赖状态** — 环境就绪 / 部分依赖缺失
- **能力统计** — 当前可用的功能模块数（满分 4/4）

### 依赖警告横幅

如果检测到 HMIRP 包缺失，首页顶部会显示醒目的警告横幅：
- 红色横幅：核心包缺失（如 HMIRP Core）
- 黄色横幅：非核心包缺失（如 Material Library）
- 蓝色横幅：可选包缺失（如 State Render System）

### 智能推荐

首页会根据当前项目状态智能推荐下一步操作：
- 零资产 + 材质库已安装 → 推荐进入 **资产浏览**
- 零资产 + Core 缺失 → 推荐进入 **车辆配置**（不依赖渲染管线）
- 有资产 + Core 已安装 → 推荐进入 **场景搭建**
- 其他情况 → 推荐进入 **资产浏览**

### 快捷入口卡片

四张功能卡片可直接跳转到对应模块：

| 卡片 | 图标 | 依赖要求 |
|------|------|---------|
| 资产浏览 | ○ | Material Library |
| 场景搭建 | ○ | HMIRP Core |
| 批量替换 | ○ | Core + Shader Library |
| 车模配置 | △ | 无（始终可用） |

> 缺失依赖的卡片会变为禁用状态并显示提示。

### 最近操作

底部显示最近 5 条操作记录，包含成功/失败状态标记。

## 资产浏览

**入口：** 侧边栏 → 资产类型（全部 / 材质库 / 模型库 / 特效库）

### 浏览模式

通过底部缩放滑条切换显示模式：

| 滑条位置 | 模式 | 说明 |
|---------|------|------|
| < 18% | 列表模式 | 仅显示名称和类型文字 |
| 18% - 100% | 网格模式 | 缩略图卡片，尺寸随滑条变化 |

卡片最小 56px，最大 260px。

### 分类筛选

- **全部** — 按材质 → 模型 → 特效 → 其他分组显示，带分隔标题
- **材质库** — 仅显示 Material 类型资产
- **模型库** — 仅显示 Model / Prefab 类型资产
- **特效库** — 仅显示 Fx 类型资产

### 搜索

顶部搜索栏支持按名称、分类、标签过滤资产，实时更新结果。

### 视图切换

顶部栏提供网格/列表视图切换按钮。

### 资产卡片

每张资产卡片显示：
- 缩略图预览（异步加载）
- 资产名称
- 来源标签
- Shader 名称（材质类型）

点击卡片可在右侧检查面板中查看详细信息。

## 批量替换

**入口：** 侧边栏 → 工具 → 批量替换

用于批量替换场景中 GameObject 上的材质。

### 操作流程

```
1. 选择目标
   在 Hierarchy 中选中需要替换材质的 GameObject

2. 查看候选
   左侧面板显示可用的候选材质网格
   使用筛选条缩小范围

3. 预览对比
   右侧面板显示替换前后的预览效果

4. 执行替换
   - "应用" — 替换单个目标
   - "全部应用" — 替换所有匹配目标
```

### 上下文信息

- 顶部显示当前目标材质名称和影响范围（受影响的对象数量）
- 支持 Undo 撤销操作

### 材质对比

点击 "对比" 按钮可进入材质对比视图，详细查看属性差异。

## 场景搭建

**入口：** 侧边栏 → 工具 → 场景搭建

基于预设模板快速生成 HMI 场景。

### 三栏布局

| 面板 | 内容 |
|------|------|
| 左栏 | 场景模板列表，点击选择模板 |
| 中栏 | 场景预览（渐变背景 + 构图参考线 + 场景名称）|
| 右栏 | 环境配置选项 |

### 环境配置

右栏提供五个配置维度：

| 配置项 | 说明 |
|--------|------|
| 照明 | 灯光方案选择 |
| 相机 | 视角与焦距配置 |
| 天气 | 天气效果预设 |
| 地面 | 地面材质与反射 |
| 天空 | 天空盒与环境光 |

### 生成

配置完成后点击 **"生成"** 按钮，工具将自动：
1. 创建新场景或在当前场景中生成
2. 放置环境元素
3. 配置灯光和相机
4. 设置天气与天空效果

> 需要 HMIRP Core 包支持。如果 State Render System 未安装，部分高级效果将不可用（界面会显示降级提示）。

## 车辆配置

**入口：** 侧边栏 → 工具 → 车辆设置

导入车辆 FBX 模型，解析零件结构，绑定材质并导出配置 Schema。

### 三栏布局

| 面板 | 内容 |
|------|------|
| 左栏 | 零件列表（按状态分组）|
| 中栏 | 3D 轨道预览（支持鼠标旋转）|
| 右栏 | 绑定配置与校验 |

### 操作流程

**第一步：导入目标**
- 将 FBX 文件拖入或通过界面选择
- 工具自动解析零件层级结构

**第二步：配置零件**
- 左栏列出所有解析出的零件
- 选中零件后，3D 预览高亮显示对应部位
- 右栏显示绑定槽位，为每个零件关联材质

**第三步：导出 Schema**
- 校验所有零件绑定状态
- 点击 **"导出"** 生成配置 Schema

### 3D 预览控制

- **旋转** — 鼠标拖拽旋转视角
- **默认视角** — 偏航 135°，俯仰 20°，距离 5

### 自动修复

如果存在校验问题，可点击 **"自动修复全部"** 尝试批量修复。

> 车辆配置不依赖 HMIRP 渲染管线，在任何环境下均可使用。

## 材质对比

**入口：** 批量替换 → 对比按钮 / 资产浏览中选择两个材质

A/B 双栏对比两个材质的属性差异。

### 布局

```
┌──────────────────────────────────────────┐
│           来源提示 · 对比模式             │
├───────────────────┬──────────────────────┤
│   A — 当前材质     │   B — 候选材质       │
│   ┌──────────┐    │   ┌──────────┐      │
│   │  预览图   │    │   │  预览图   │      │
│   └──────────┘    │   └──────────┘      │
│   名称             │   名称              │
│   Shader           │   Shader            │
│   属性列表         │   属性列表          │
├───────────────────┴──────────────────────┤
│             差异摘要                      │
└──────────────────────────────────────────┘
```

### 进入方式

- 从 **批量替换** 进入：自动填充当前材质和候选材质
- 从 **资产浏览** 进入：需选中两个材质资产

```
│ · 资产  │  · 首页仪表盘            │              │
│ · 工具  │  · 资产网格              │              │

├────────┴──────────────────────────┴──────────────┤
│                   底部栏                         │

└─────────────────────────────────────────────────┘
```

### 侧边栏

- **导航区** — 首页按钮
- **资产类型** — 全部、材质库、模型库、特效库（显示实时计数）
- **工具区** — 车辆设置、批量替换、场景搭建
- **环境状态** — HMIRP 四包安装状态摘要

### 顶部栏

- 左侧：产品标题、当前选中对象、活动场景
- 中部：搜索栏
- 右侧：视图模式切换、渲染管线标签

- 左侧：状态指示灯（绿色成功/橙色失败/灰色正常）+ 操作消息
- 右侧：撤销按钮、操作计数、渲染管线信息

### 检查面板（右栏）

根据当前工作流自动切换内容：

| 模式 | 面板内容 |
|------|---------|
| 资产浏览 | 三步引导：选择目标 → 选择资产 → 应用操作 |
| 批量替换 | 替换上下文和操作面板 |
| 其他模式 | 通用信息展示 |

## 常见问题

### Q: 打开后显示"需要安装 HMIRP 渲染管线"

A: 需要在项目中安装 HMIRP Core 包。支持三种安装方式：
- Package Manager 直接搜索安装
- manifest.json 中添加 git URL
- 本地 `file:` 路径引用

工具会自动从 git URL 的 `#release/x.y.z` 标签中识别版本号。

### Q: 资产缩略图显示为黑色

A: 缩略图通过 Unity 的 `AssetPreview.GetAssetPreview()` 异步生成，首次加载可能需要等待。如果持续为黑色：
- 确认资产文件未损坏
- 尝试在 Project 窗口中选中资产触发预览生成
- 重新打开工作区窗口

### Q: 部分功能卡片显示为禁用状态

A: 功能依赖特定的 HMIRP 包。查看卡片底部的禁用提示（如"需安装 HMIRP Core"），安装对应包后自动激活。车辆配置不依赖任何 HMIRP 包，始终可用。

### Q: 如何撤销操作

A: 点击底部栏的 **撤销** 按钮，或使用标准 Unity 快捷键 `Ctrl+Z` / `Cmd+Z`。

### Q: 环境状态显示"4/4 能力可用"但仍有警告

A: 关闭并重新打开工作区窗口以刷新依赖检测状态。
